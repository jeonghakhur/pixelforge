# Design Context — PixelForge

## Target Audience
- **Primary**: Frontend/Design System engineers managing Figma design tokens
- **Secondary**: Designers verifying token-to-code consistency
- **Context**: Internal tool, used daily — not a marketing page

## Use Cases
- Extract tokens from Figma (colors, spacing, radius, typography)
- Browse and search token values by type
- Inspect sync history and token diffs
- Generate Bootstrap component code from tokens
- Verify token changes before committing to design system

## Brand Personality / Tone
**Vantablack Luxe** — dark IDE aesthetic with precision and restraint.
- Deep black base (`$zinc-950`)
- Single accent color: blue (`$blue-500` dark / `$blue-600` light)
- Monospace type for data/labels, thin sans-serif for headings
- No gradients, no glows, no "AI slop" purple/cyan
- Feels like a high-end code editor, not a dashboard SaaS

## Technical Constraints
- SCSS Modules (no Tailwind, no inline styles)
- CSS custom properties only — no hardcoded hex in component SCSS
- 8px spacing scale (`$spacers` map)
- Solar icon set via `@iconify/react`
- WCAG AA contrast (4.5:1 minimum)

## Quality Bar
**Flagship** — this is the main working surface, not an MVP.
Every detail matters: spacing rhythm, interaction states, reduced motion.

## What Makes This Unforgettable
The dot-grid canvas background + thin-weight large numerics + mono uppercase labels.
The IDE chrome feeling — not a dashboard, a precision instrument.

## Design System
- Tokens: `src/styles/_variables.scss`
- Global CSS vars: `src/styles/globals.scss`
- Mixins: `src/styles/_mixins.scss`
- Key vars: `--accent`, `--text-primary/secondary/muted`, `--bg-surface/elevated`, `--glass-border/bg`, `--success/danger/warning` (+ `-subtle` variants)

## Design Context

### Users
디자이너와 프론트엔드 개발자가 함께 사용하는 협업 도구. 디자이너는 Figma에서 Variables/Styles를 내보내고, 개발자는 IDE처럼 생긴 이 앱에서 토큰을 확인하고 컴포넌트 코드를 생성한다. 두 직군 모두 도구에 익숙하고 밀도 높은 정보 화면을 선호한다.

**핵심 작업(JTBD):** "Figma 디자인을 코드로 정확히 옮기는 과정에서 발생하는 불일치를 없앤다."

### Brand Personality
**정밀 · 전문 · 절제**

- 정밀: 픽셀 단위의 정확성. 반올림 없는 수치, 정렬된 간격, 일관된 레이어
- 전문: IDE에서 일하는 사람의 도구. 장식보다 기능, 과시보다 신뢰
- 절제: 불필요한 색, 그림자, 애니메이션 없음. 필요한 것만 정확하게

감정 목표: 사용할 때 "잘 만들어진 도구를 쓰는" 안정감과 통제감.

### Aesthetic Direction
**Vantablack Luxe** — 순수한 다크 테마, 고급스러운 어두움

- 베이스: `#09090b` (zinc-950). 순수 검정(`#000000`) 금지
- 액센트: `#3b82f6` (blue-500) 단 하나. 보라/파란 그라디언트, AI 글로우 금지
- 글래스모피즘: `rgba(255,255,255,0.05)` 배경 + `rgba(255,255,255,0.08)` 테두리로 레이어 표현
- 레이아웃: 비대칭 Bento Grid 선호. 3열 균등 카드 금지
- 트랜지션: 항상 스프링 커브 `cubic-bezier(0.16, 1, 0.3, 1)`. `linear`/`ease-in-out` 금지

**안티 레퍼런스:**
- Notion: 밝고 문서 중심적인 느낌 → PixelForge는 어둡고 밀도 높음
- Bootstrap 기본: 무색의 기능적 컴포넌트 → PixelForge는 테마가 강하고 정제됨
- AI 스타트업: 보라/파란 그라디언트, 과한 글로우 → 절대 금지

### Design Principles

1. **정보 밀도 우선** — 스크롤을 줄이고 한 화면에 더 많은 컨텍스트를 담는다. 여백은 호흡을 위한 것이지 채우기 위한 것이 아니다.

2. **단 하나의 액센트** — `var(--accent)` (blue-500)만 CTA와 활성 상태에 사용한다. 두 번째 강조색이 필요하다면 시맨틱 색(danger/success/warning)을 쓴다.

3. **레이어로 깊이 표현** — 그림자 대신 배경색의 밝기 차이(`bg-body → bg-surface → bg-elevated`)로 층위를 만든다.

4. **아이콘은 Solar 세트만** — `solar:*-linear` 또는 `solar:*-bold`. Lucide, FontAwesome 금지.

5. **코드처럼 읽히는 UI** — 모노스페이스 폰트(Geist Mono)를 수치, 해시, 버전, 코드 값에 적극 사용한다. 숫자가 정렬되면 전문성이 드러난다.
