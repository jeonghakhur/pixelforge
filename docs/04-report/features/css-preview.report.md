## 3. Implementation Details

### 3.1 Core Components

#### CssPreviewModal.tsx (178 lines)
**위치**: `src/app/(ide)/tokens/[type]/CssPreviewModal.tsx`

**역할**: 다크 모달에서 CSS 코드를 테이블 형식으로 표시

**주요 기능**:
```tsx
- createPortal 다크 오버레이 (88vw × 88vh)
- 비동기 로딩: getAllTokensAction() → 모든 토큰 가져오기
- useMemo: generateAllCssCode(tokens) → CSS 코드 생성
- useMemo: hljs.highlight(line, { language: 'css' }) → 라인별 강조
- 복사 버튼: navigator.clipboard.writeText(cssCode)
- 다운로드 버튼: Blob + URL.createObjectURL (新)
- 포커스 관리: closeRef.current?.focus() on open
- ESC 키: document.addEventListener('keydown', e => e.key === 'Escape')
- Body scroll lock: document.body.style.overflow = 'hidden'
```

**접근성 준수**:
- ✅ `role="dialog"`, `aria-modal="true"`
- ✅ ESC 닫기, body 스크롤 차단
- ✅ 다운로드/복사 버튼 `aria-label`
- ✅ 코드 영역 `tabIndex={0} role="region" aria-label="CSS 코드"`

#### css-generator.ts (154 lines)
**위치**: `src/lib/tokens/css-generator.ts`

**함수 1**: `generateCssCode(tokens, type): string`
- 특정 타입(color/typography/spacing/radius)의 토큰만 처리
- `TokenRow[]` → `:root { --color-*, ... }`

**함수 2**: `generateAllCssCode(allTokens): string` (新)
- 모든 타입의 토큰을 단일 `:root { }` 블록으로 통합
- 타입별 섹션 헤더: `/* ── Color ── */`
- 그룹 내 토큰들 정렬: 가장 긴 변수명 기준 패딩

**변수명 규칙**:
```
tokenName: "brand/primary"
type: "color"
        ↓
--color-brand-primary
        ↑         ↑
    prefix    (slash → hyphen, lowercase)
```

**출력 예**:
```css
/* Generated by PixelForge · 2026-03-24 */

:root {
  /* ── Color ── */
  /* brand */
  --color-brand-primary:         #007AFF;
  --color-brand-primary-hover:   #0066D6;

  /* ── Typography ── */
  /* heading */
  --font-heading-h1:             24px 700;
  ...
}
```

## 7. Lessons Learned

### 7.1 What Went Well

1. **Direct highlight.js 접근법** — diff2html 대신 직접 per-line 테이블로 렌더링하면서 불필요한 렌더링 오버헤드 제거
   - 결과: 더 빠른 로딩, 더 깔끔한 UI

2. **모든 토큰 통합 출력** — 단일 CSS 파일로 모든 디자인 토큰 관리 가능
   - 개발자 워크플로우: 한 파일만 복사/다운로드하면 모든 변수 추가 완료

3. **createPortal + dark overlay** — Bootstrap Modal 대신 자체 포탈로 GitHub 스타일 유지
   - CodeViewModal과 일관된 aesthetic 제공
   - 사용자가 이미 친숙한 패턴

4. **자체 상태 관리 (TokenPageActions)** — prop drilling 제거
   - 캡슐화 개선, 재사용성 높음

5. **접근성 처음부터 내장** — ESC, 포커스, ARIA 모두 구현 단계에서 처리
   - 나중에 수정할 기술 부채 없음

### 7.2 Areas for Improvement

1. **CSS 검증 로직 부재** — 생성된 CSS가 유효한지 검사 안 함
   - 향후: CSS parser로 검증 추가

2. **포맷 옵션 없음** — CSS Variables 외 SCSS/JS/Tailwind 선택 불가
   - 향후: Format tabs로 확장

3. **Diff view 미구현** — before/after 토큰 변경 비교 불가
   - 향후: `wrapAsDiff()` 활용

4. **Performance** — 토큰 수 많을 때 라인 강조가 동기 (useMemo)
   - 향후: Web Worker로 비동기 처리

### 7.3 To Apply Next Time

- **Integrate accessibility from Day 1** — 나중에 추가하지 말고 처음부터 WCAG AA 준수
- **Test with real data** — 더미 데이터 아닌 실제 색상/타이포 토큰으로 테스트
- **User-facing UI 먼저 확인** — 버튼/모달 위치, 텍스트, 아이콘이 UX에 미치는 영향 크다
- **Reuse GitHub dark theme** — CodeViewModal처럼 이미 검증된 aesthetic 활용
